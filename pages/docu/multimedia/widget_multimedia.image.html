/** * ----------------------------------------------------------------------------- * @package smartVISU * @author Martin Glei√ü * @copyright 2012 - 2015 * @license GPL [http://www.gnu.de] * ----------------------------------------------------------------------------- */ {% extends "widget_multimedia.html" %} {% block header %} {% endblock %} {% block example %} The widget displays images and streams which are accessible via an URL - nearly everything that can be displayed by putting it's URL into an <img> tag. Some browsers reject URLs containing credentials. For streams containing pure JPEG images, this can be avoided by setting 'localize' to true even with high fps rates. The localize method does not work on MJPEG streams.
To display images which come as Base64 coded data in an item, see basic.print.
Examples
The first example uses an item called "webcam_item" which holds the image URL. A second item ("refresh_item") is used to trigger the refreshing of the image with every item update. Use the switch to refresh the image:
{{ basic.flip('', 'refresh_item') }}
The image may change only slightly. Inspect the element using the browsers developer tools to see the timestamp changing behind the image name.
